3. DESIGN TOKENS
================================================================================
COLOR ROLES (reference is black/white/gray minimalist; re-skin to Clickpost.
Use Clickpost brand colors — values below are a sensible starting palette,
tune to the official brand kit):

  --background        #FFFFFF   (page base)
  --background-muted  #F6F7F9   (alternating section bg, stats band)
  --foreground        #0B0B0F   (headings — near-black)
  --muted-foreground  #5A6070   (body text, secondary)
  --border            #E6E8EC
  --primary           #2F5BFF   (Clickpost accent — CTAs, links, highlights)
  --primary-foreground#FFFFFF
  --accent-soft       #EAF0FF   (tinted chips / icon backgrounds)

  Buttons in reference are solid-black; map "solid" to --primary, keep a
  high-contrast solid + an outline/ghost secondary variant.

TYPOGRAPHY:
  Font: clean geometric/grotesque sans (e.g. Inter, or Clickpost brand font
  via next/font). Define scale:
    Display (hero h1)  : clamp(2.5rem, 5vw, 4rem),  weight 700, tight tracking, line-height ~1.05
    H2 (section title) : clamp(2rem, 3.5vw, 3rem),  weight 700
    H3 (card title)    : 1.25–1.5rem, weight 600
    Eyebrow / label    : 0.8rem, weight 600, UPPERCASE, letter-spacing 0.08em, --primary or --muted-foreground
    Body               : 1rem–1.125rem, weight 400, line-height 1.6
    Small / caption    : 0.875rem
  High contrast between bold headings and regular body. Quotes can be italic.

SPACING / RHYTHM:
  - Section vertical padding: py-20 (mobile) → py-28/py-32 (desktop). Generous.
  - Container max-width: ~1200–1280px, horizontal gutter px-6 (mobile) / px-8.
  - Consistent 8px spacing base; gaps of 16/24/32/48/64/80/96/128.

RADIUS:
  - Buttons & cards: rounded-xl (12px). Pills/chips: rounded-full.
  - Images/screenshots: rounded-2xl (16px) with subtle border.

SHADOWS / ELEVATION:
  - Cards: subtle — shadow-sm to shadow-md, soft, low-spread, low-opacity.
  - Floating product imagery: larger soft shadow (shadow-xl, blurred).
  - Sticky nav: faint shadow / border appears only after scroll.

4.4 SIGNATURE BACKGROUND — SCROLL-DRIVEN 3D GLOBE  [GlobeBackground.tsx]
================================================================================
Ambient 3D globe in the background that REACTS TO SCROLL (inspired by the
redo.com homepage globe). It is the site's signature visual motif. Decorative
only (aria-hidden) — never carries essential content.

TECH:
  - three.js via @react-three/fiber (+ @react-three/drei), OR react-globe.gl.
    Prefer react-three-fiber for full control of scroll-linked camera/material.
  - Rendered in a FIXED/PINNED background layer behind page content (z-index
    below text, above page bg). pointer-events: none.
  - Dynamic import with { ssr: false }; lazy-mount; render loop only when the
    canvas is on-screen (IntersectionObserver) and tab is visible.

IDLE BEHAVIOR:
  - Slow continuous auto-rotation + gentle floating bob.
  - Subtle atmosphere/glow rim in --primary (brand color) tint.
  - Soft directional + ambient lighting; faint star/particle field optional.

SCROLL-DRIVEN BEHAVIOR (the key effect):
  - Use Framer Motion useScroll (page progress) + useTransform to map scroll
    progress → globe rotation (Y/X), tilt, zoom (camera distance / scale),
    and on-screen X/Y offset. Apply per-frame via useFrame (lerp/damp for
    smoothness, never snap).
  - Choreography by section, e.g.:
      Hero            : centered-ish, large, slow spin behind the headline+demo
      Mid sections    : drifts to one side, slightly smaller, faster spin
      "Global reach"  : scales up / re-centers, dots + arcs emphasized
      Footer approach : settles, dims down
  - PRODUCT NARRATIVE (DECIDED — trial, may change): arcs = "orders saved",
    not shipments. Each arc represents a shopper self-serving instead of
    contacting support. As an arc lands, a small label/tooltip fires:
      "Address corrected — Austin, TX" / "Order edited — Lisbon" /
      "Upsell accepted — Berlin" / "Cancellation avoided — Mumbai"
    Pair with a LIVE COUNT-UP stat that ticks as arcs land
      ("X edits handled · $Y upsell revenue · Z cancellations recovered").
    Use placeholder/illustrative numbers until real ones are supplied.
  - Glowing location DOTS + animated ARCS on a loop.

PERFORMANCE:
  - Cap devicePixelRatio (e.g. min(window.dpr, 2)); throttle on mobile.
  - Pause render when offscreen / tab hidden. Dispose geometries on unmount.
  - Keep texture sizes modest; compress. Target 60fps desktop, no jank.

FALLBACKS:
  - prefers-reduced-motion, low-power, or small/mobile screens → render a
    STATIC globe image or a CSS radial-gradient glow instead of WebGL.
  - If WebGL unavailable → graceful static fallback, no errors.

A11Y:
  - aria-hidden="true", not focusable, decorative only. All copy/contrast must
    pass independently of the globe (ensure text remains legible over it —
    add a subtle scrim/blur behind text where they overlap).

4.5 FLAGSHIP FEATURE — "DEMO FROM YOUR STORE URL"  [LiveDemoGenerator.tsx]
================================================================================
This is the hero/standout feature (inspired by redo.com's store-URL demo
flow, e.g. redo.com/search/brands/{store}?products=order-editing). It must be
the most polished, best-animated part of the site. Make it visual-first.

CONCEPT:
  User pastes their store URL → we generate a PERSONALIZED, branded demo that
  shows the CP: Order Editing & Upsell experience running on THEIR own store's
  branding + real products. Zero setup, instant "wow".

PLACEMENT (DECIDED):
  - Input lives IN THE HERO, front-and-center ("See it on your store — paste
    your URL"). This is the headline feature. Submitting reveals the generated
    demo directly below the hero with a smooth scroll/expand transition.

BUILD MODE (DECIDED):
  - MOCK DATA FOR NOW. Build the full UI + animation + interactive widget using
    hardcoded sample store data (a realistic fake store: logo, brand color, a
    few products w/ placeholder images). Structure the code so the real
    /api/demo fetch (Shopify products.json + OG branding) can be wired in later
    with no UI changes — keep a single data-provider seam (e.g. useDemoData()
    that currently returns the mock and will later call the route).

UX FLOW:
  1. Input field + "Generate demo" button. Validate URL; accept bare domains
     (judo.com) and full URLs. Show example/placeholder text.
  2. On submit: optimistic loading state — branded skeleton + playful status
     copy ("Reading your store…", "Pulling your products…", "Building your
     demo…"). Staggered shimmer. Keep it ~1–2s even if fast (feels crafted).
  3. Reveal: an interactive MOCK order-status / thank-you page, re-skinned with
     the store's logo + brand color, populated with the store's real products
     and imagery, with the CP widget live on top.
  4. Interactive widget actions the visitor can click through:
       - Edit items / quantities / variants
       - Change shipping address (Google-Maps-style validation UI)
       - Cancel order → refund / store-credit options
       - Post-purchase UPSELL card (recommends another of their products)
     Each action animates (modal/drawer, success toasts) — fully clickable,
     no real mutations (sandbox/mock state only).

DATA SOURCING (Next.js Route Handler, server-side):
  - POST /api/demo  { url }  → returns { brand: {name, logo, color}, products[] }
  - Products + images: fetch the store's PUBLIC Shopify JSON:
        https://{store}/products.json?limit=12   (and /collections/all/products.json)
  - Brand: fetch the homepage HTML, parse Open Graph (og:image, og:site_name)
    + <link rel="icon"> for logo, and sample a theme/brand color (meta
    theme-color, or derive from logo). Cache results (short TTL).
  - Sanitize + proxy remote images via next/image remotePatterns or an image
    proxy route to avoid hotlink/CORS issues.
  - Rate-limit the route; timeout fetches (~4s); never block the UI thread.

FALLBACKS (must never break):
  - Non-Shopify / blocked / no products / error → show a polished GENERIC demo
    using neutral placeholder branding + placeholder products, with a small
    note ("Showing a sample demo"). The flow still fully works.
  - prefers-reduced-motion → skip shimmer/parallax, keep instant reveal.

ANIMATION (best-in-class):
  - Input → results: smooth height/scroll transition (Framer Motion layout).
  - Branded reveal: logo/color "paint-in", products stagger-fade into a grid.
  - Widget actions: spring-based drawers/modals, micro success animations.
  - Optional device/browser frame around the mock for realism.

ACCESSIBILITY:
  - Input has a real <label>; button announces loading (aria-busy/live region).
  - Mock interactions keyboard-operable; focus moves into revealed demo.

COMPONENTS:
  LiveDemoGenerator.tsx (orchestrator), DemoUrlInput.tsx, DemoSkeleton.tsx,
  DemoStorefront.tsx (branded mock), OrderEditWidget.tsx (the CP widget mock),
  UpsellCard.tsx, AddressValidator.tsx. Mock state via a small reducer/hook.

5. ANIMATION & INTERACTION SYSTEM (match the reference's quality)
================================================================================
GLOBAL MOTION PRINCIPLES:
  - Easing: easeOut / custom cubic-bezier(0.16, 1, 0.3, 1) for entrances.
  - Durations: 0.4–0.7s entrances; 0.15–0.25s hover/tap feedback.
  - Respect prefers-reduced-motion → disable transforms, keep opacity only.

SCROLL REVEALS:
  - Sections/elements fade-up (opacity 0→1, y 24→0) when entering viewport.
  - Use Framer Motion `whileInView` + `viewport={{ once: true, margin }}`.
  - Stagger children within a section for a cascading reveal.

HERO:
  - Staggered load-in. Optional subtle parallax on the product image as you
    scroll (useScroll + useTransform).

MARQUEES / CAROUSELS:
  - Infinite loop, constant speed, pause on hover.

HOVER MICRO-INTERACTIONS:
  - Buttons: slight scale (1.0→1.02) + bg/shade shift, 0.2s.
  - Cards: translateY(-4px) + shadow increase + optional border→primary.
  - Logos: grayscale→color, opacity 0.6→1.
  - Links: animated underline (left-to-right) or color shift + arrow nudge (→).

NAV:
  - Scroll-aware bg/shadow/shrink transition.
  - Dropdown mega-menus fade+slide.

NUMBERS:
  - Count-up animation when stats scroll into view.

PAGE TRANSITIONS:
  - Subtle fade between route changes (Framer Motion AnimatePresence) if
    multi-page; otherwise smooth in-page anchor scrolling.
